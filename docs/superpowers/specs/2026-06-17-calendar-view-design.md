# 日历视图 — 设计文档

## 概述

新增日历视图页面，以 iOS 风格月历展示每日收支，支持快速浏览和跳转流水明细。

## 导航与路由

- 底部导航栏新增「日历」Tab（📅），位于「记账」和「流水」之间
- 路由：`/calendar`
- 新建 `src/pages/Calendar.tsx`，通过 React.lazy 代码分割加载

### 月份选择

两级选择：先选年份、再选月份。

- 年份选项：当年 + 后 5 年，共 6 年
- 月份选项：1-12 月
- 默认选中当月

### 星期头

周一至周日，固定显示。与项目现有工具函数 `getDayOfWeek` 保持一致。

### 日期格子

- iOS 风格大方格，每格约 48-52px 高
- 日期数字在格子右上角，小字
- 收入汇总在上（绿色 `text-income`）、支出汇总在下（红色 `text-expense`），均使用 `font-mono tabular-nums`
- 无收支的格子只显示日期数字，不显示金额
- 非当月日期（填充格）：灰色文字，不显示金额
- 今天：蓝色背景高亮

## 交互

- 点击有收支的日期 → 跳转 `/records?date=YYYY-MM-DD`
- 点击无收支的日期 → 无反应

## 数据流

```
Calendar 页面
  ├── useTransactions()           → 全部交易数据（不传 month 参数）
  └── 组件内计算
       ├── 按 date 字段分组，汇总每日 expense/income 总额
       ├── 生成该月日历网格（含前后月填充日，用于补齐首尾周）
       └── 跳转时拼接 URL: /records?date=YYYY-MM-DD
```

## Records 页面改动

从 URL 读取 `date` 查询参数（`useSearchParams`）：

- `date` 存在时：只显示该日期流水，隐藏月份下拉和分类/支付筛选，显示日期标题 + "← 返回整月" 链接
- `date` 不存在时：保持现有按月筛选行为
