# 日历页设计改版规范

> 2026-06-22 | 状态：待实现

## 目标

将日历页从纯数据网格改造为"消费节奏热力图"，极简生活 App 风格，与 Dashboard 互补。

## 页面结构（自上而下）

1. **月份标题 + 下拉导航** — 保留原 select 下拉框选择年/月
2. **月度摘要条** — 日均支出 / 月累计 / 交易天数，各带环比箭头
3. **星期头** — 一～日
4. **热力图网格** — 35-42 格，背景色编码支出强度
5. **色阶图例** — 渐变色条 ¥0 → ¥200+
6. **详情面板** — 点击日期后底部展开，分类明细 + 日均对比

## 热力图色阶（固定阈值，5 档）

| 档位 | 阈值（元） | 阈值（分） | 背景色 | 文字色 |
|------|----------|-----------|--------|--------|
| L0 无支出 | ¥0 | 0 | `#fff` + `border: #f0ede7` | `#333` |
| L1 微量 | ¥1 – 49 | 1 – 4,900 | `#fef2f2` | `#dc2626` |
| L2 轻度 | ¥50 – 99 | 5,000 – 9,900 | `#fee2e2` | `#dc2626` |
| L3 中度 | ¥100 – 199 | 10,000 – 19,900 | `#fecaca` | `#991b1b` |
| L4 重度 | ≥ ¥200 | ≥ 20,000 | `#fca5a5` | `#7f1d1d` |

- 使用固定阈值确保跨月可比性
- 所有色值使用 Tailwind red 色系，不引入新的调色板依赖

### 特殊状态标记

| 状态 | 视觉处理 |
|------|----------|
| 今日 | 蓝色 ring `box-shadow: 0 0 0 2px #3b82f6` |
| 收入日 | 日期数字左上角 5px 绿色圆点 `#10b981` |
| 非当月 | 灰色背景 `#f0ede7`，浅灰文字 |
| 选中态 | 蓝色 ring + scale(1.05) + 其他格子 opacity: 0.5 |

## 月度摘要条

白色卡片，三分栏布局，竖线分隔：
- **日均支出**：`¥86` + 较上月 `▲12%`（红色）
- **月累计**：`¥1,892` + 较上月同期 `▲8%`（红色）
- **交易天数**：`18/22` + "22天有记录"

环比计算规则：
- 日均：本月日均 vs 上月日均
- 月累计：本月累计 vs 上月同期（例如6月前22天 vs 5月前22天）

## 详情面板

### 触发
- **点击**有交易的日期 → 底部滑入面板
- **长按**日期 → 跳转 `/records?date=YYYY-MM-DD`
- 无支出日期点击无反应

### 面板内容
- 拖拽指示条（灰色横条）
- 日期 + 星期（如"6月7日 · 周六"）
- 当日总支出金额
- "长按日期格跳转查看全部记录" 提示
- 分类明细列表，每行：图标 + 分类名 + 金额 + 较日均差（▲32% / ▼15% / —）
- 日均差只在该分类当前月的日均消费 > 0 时显示

### 关闭
- 再次点击同一日期
- 点击面板外空白区域
- 点击另一日期（切换面板内容）

### 动画
- 展开：`slide-up` 动画（已有）, 0.3s ease-out
- 选中格：scale(1.05), 150ms ease-out
- 切换月份：标题淡入淡出

## 边缘状态

- **当月无交易**：热力图全白 + EmptyState "这个月还没有记录"
- **加载中**：热力图骨架屏（灰色方块占位）
- **金额极大**：超出 L4 阈值仍用 L4 颜色，格内数字截断为 `¥999+`

## 数据计算

需要在 `useMemo` 中新增：
- `heatLevel(date)` — 按当日支出总额返回 0-4 档位
- `dailySummary` — 已有，复用
- `categoryDailyAvg(categoryId)` — 某分类本月日均
- `monthSummary` — 本月日均、月累计、上月同期对比、交易天数

## 不改动

- 不新增 npm 依赖
- 不修改数据模型
- 不修改 adapter 层
- 不修改路由结构
- 保留现有的 `getCalendarDays()` 工具函数
